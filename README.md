# 創鈺規範
所謂『無規不成圓，無矩不成方』，一個團隊在合作開發時大家總要遵循一些總則，才不至於每個人寫的樣子完全的不一樣，到最後互相維護時才發現根本就看不懂

當然不是說要用規範來限制開發人員，基本上只會設定大方向原則

以下撰寫出相關原則，若有問題，大家可以利用 GitHub 所提供的相關功能一起來編修

```
＊ 單純疑問可開 issue 出來
＊ 有相關建議編輯可以使用 pull request 來協同編輯
```
